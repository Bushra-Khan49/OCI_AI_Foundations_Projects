train_eval.py

Standalone, reproducible comparison of linear vs. non-linear classifiers on
the Golub et al. (1999) leukemia gene expression dataset (ALL vs. AML).

While Project 1 demonstrated that neural networks easily outperform linear models on 
synthetic 2D data, biological data is often the exact opposite. With 3,571 genes and only 
72 samples (patients), we are in a high-dimensional regime (p >> n). In this space, 
complex models like neural networks are prone to severe overfitting, and classical linear 
models usually act as a much stronger baseline.

Run `python data/download_data.py` first to produce data/leukemia_clean.csv.

Usage:
    python train_eval.py
    python train_eval.py --n-features 100 --n-splits 10 --results-dir results\n\n\n## Step 1: Import libraries and setup CLI\nFirst, I import the necessary libraries for data processing, machine learning\npipelines, and visualization.\n\n```python\nimport argparse\nimport json\nfrom pathlib import Path\n\nimport numpy as np\nimport pandas as pd\nimport matplotlib\nmatplotlib.use("Agg")\nimport matplotlib.pyplot as plt\n\nfrom sklearn.decomposition import PCA\nfrom sklearn.preprocessing import StandardScaler\nfrom sklearn.feature_selection import SelectKBest, f_classif\nfrom sklearn.pipeline import Pipeline\nfrom sklearn.model_selection import StratifiedKFold, cross_val_score\nfrom sklearn.linear_model import LogisticRegression\nfrom sklearn.svm import SVC, LinearSVC\nfrom sklearn.neural_network import MLPClassifier\nfrom sklearn.dummy import DummyClassifier\n\n\ndef parse_args():\n    # Setup argparse for CLI usage\n    p = argparse.ArgumentParser(description="Linear vs. non-linear classifiers on Golub leukemia data.")\n    p.add_argument("--data-path", type=str, default="data/leukemia_clean.csv")\n    p.add_argument("--n-features", type=int, default=50, help="Genes to keep via ANOVA F-test, per fold.")\n    p.add_argument("--n-splits", type=int, default=5, help="Number of CV folds.")\n    p.add_argument("--random-state", type=int, default=0)\n    p.add_argument("--results-dir", type=str, default="results")\n    return p.parse_args()\n\n\ndef main():\n    # Parse CLI arguments and setup results directory\n    args = parse_args()\n    results_dir = Path(args.results_dir)\n    results_dir.mkdir(parents=True, exist_ok=True)\n\n    data_path = Path(args.data_path)\n    if not data_path.exists():\n        raise FileNotFoundError(\n            f"{data_path} not found. Run `python data/download_data.py` first."\n        )\n\n```\n\n## Step 2: Load Data and Establish Baseline\nI load the cleaned dataset. It contains 72 patients (our samples) and\n3,571 genes (our features), labeled as either ALL or AML leukemia.\n\nBefore training any model, I establish a 'dummy' baseline. This model simply\nguesses the most frequent class (ALL) every single time. It achieves about 65%\naccuracy. This establishes the absolute minimum floor that a real model has\nto beat to prove it is learning actual biological signals.\n\n```python\n    # Load dataset into pandas\n    df = pd.read_csv(data_path)\n    # Extract feature matrix (genes)\n    X = df.drop(columns=["label"]).values\n    # Extract binary target vector (AML=1, ALL=0)\n    y = (df["label"] == "AML").astype(int).values\n\n    print(f"Loaded {X.shape[0]} samples, {X.shape[1]} genes.")\n    print(f"Class balance: ALL={np.sum(y==0)}, AML={np.sum(y==1)}")\n\n    # Because I only have 72 samples, doing a single 70/30 train/test split would leave \n    # only ~21 patients in the validation cohort. That is far too few to trust the results.\n    # I use 5-fold cross-validation (CV) instead.\n    cv = StratifiedKFold(n_splits=args.n_splits, shuffle=True, random_state=args.random_state)\n\n    # Initialize a naive model that only predicts the majority class\n    dummy = DummyClassifier(strategy="most_frequent")\n    # Evaluate naive model to establish a performance floor\n    dummy_scores = cross_val_score(dummy, X, y, cv=cv, scoring="accuracy")\n    print(f"\nMajority-class baseline: {dummy_scores.mean():.3f} +/- {dummy_scores.std():.3f}")\n\n\n```\n\n## Step 3: PCA to 2D for visual inspection\nBecause of human limitations, we can only visually perceive up to 3 dimensions.\nWe cannot plot 3,571 genes on a graph. To overcome this, I use Principal\nComponent Analysis (PCA) to compress all 3,571 genes down to 2 dimensions.\nI do this purely for a visual sanity check to see if the ALL and AML patients\nnaturally cluster apart based on their gene expressions. I do not train the\nmodels on this 2D data.\n\n```python\n    # Scale features for PCA to prevent large magnitude features from dominating\n    X_scaled = StandardScaler().fit_transform(X)\n    # Compress 3571 features down to 2 Principal Components\n    pca = PCA(n_components=2, random_state=args.random_state)\n    X_pca = pca.fit_transform(X_scaled)\n\n    # Plot PCA\n    fig, ax = plt.subplots(figsize=(6, 5))\n    for label, name, color in [(0, "ALL", "tab:blue"), (1, "AML", "tab:orange")]:\n        mask = y == label\n        ax.scatter(X_pca[mask, 0], X_pca[mask, 1], label=name, alpha=0.7, color=color)\n    ax.set_xlabel(f"PC1 ({pca.explained_variance_ratio_[0]*100:.1f}% var)")\n    ax.set_ylabel(f"PC2 ({pca.explained_variance_ratio_[1]*100:.1f}% var)")\n    ax.set_title("PCA projection: ALL vs. AML gene expression")\n    ax.legend()\n    fig.tight_layout()\n    fig.savefig(results_dir / "pca_projection.png", dpi=150)\n    plt.close(fig)\n\n```\n\n## PCA Projection Interpretation:\nLooking at the generated pca_projection.png, we can see that the ALL (blue)\nand AML (orange) patients form somewhat distinct clusters, but there is noticeable\noverlap. This indicates that while there is strong biological signal, the classes\naren't trivially separable in 2D space. The models will need to leverage the\nhigher-dimensional space.\n\n\n## Step 4 & 5: Setup Pipeline, CV, and Compare Models\nNow, I compare a suite of linear models against non-linear models.\n\nIdentical Preprocessing & Information Leaks:\nTo ensure a fair, apples-to-apples comparison, I use a scikit-learn `Pipeline`.\nFeature selection (keeping the top 50 genes via ANOVA F-test) MUST happen\n*inside* the cross-validation loop. If I filtered the top 50 genes using the\nentire 72 patients before splitting the data, the training phase would get an\nunfair 'sneak peek' at the validation cohort. This is an information leak.\n\n```python\n    def make_pipeline(clf):\n        # Create an identical pipeline for every model\n        return Pipeline([\n            ("scale", StandardScaler()),\n            ("select", SelectKBest(score_func=f_classif, k=args.n_features)),\n            ("clf", clf),\n        ])\n\n    # Define linear and non-linear models\n    models = {\n        "logistic_regression": LogisticRegression(max_iter=5000),\n        "linear_svm": LinearSVC(max_iter=5000),\n        "rbf_svm": SVC(kernel="rbf"),\n        "mlp_10_hidden": MLPClassifier(hidden_layer_sizes=(10,), max_iter=3000, random_state=args.random_state),\n    }\n\n    cv_results = {}\n    print(f"\nComparing models with {args.n_splits}-fold CV, top {args.n_features} genes per fold:")\n    for name, clf in models.items():\n        # Evaluate model securely using cross-validation\n        pipe = make_pipeline(clf)\n        scores = cross_val_score(pipe, X, y, cv=cv, scoring="accuracy")\n        cv_results[name] = {"mean": float(scores.mean()), "std": float(scores.std()), "folds": scores.tolist()}\n        print(f"  {name:25s}  {scores.mean():.3f} +/- {scores.std():.3f}")\n\n\n```\n\n## Step 6 & 7: Plot the comparison & Interpretation\nI visualize the cross-validation accuracy of the models using a bar chart.\n\nAs anticipated, in this high-dimensional p >> n setting, the linear models\n(Logistic Regression, Linear SVM) perform just as well as, or slightly better\nthan, the complex non-linear models (MLP). The neural network shows higher\nvariance (wider error bars) and lower mean accuracy because its excess capacity\nmakes it prone to overfitting the 72 samples.\n\n```python\n    fig, ax = plt.subplots(figsize=(7, 4))\n    names = list(cv_results.keys())\n    means = [cv_results[n]["mean"] for n in names]\n    stds = [cv_results[n]["std"] for n in names]\n    # Draw horizontal bar chart\n    ax.barh(names, means, xerr=stds, color="tab:blue", alpha=0.8)\n    # Overlay the baseline performance\n    ax.axvline(dummy_scores.mean(), color="red", linestyle="--",\n               label=f"Majority-class baseline ({dummy_scores.mean():.2f})")\n    ax.set_xlabel(f"{args.n_splits}-fold CV accuracy")\n    ax.set_xlim(0, 1.05)\n    ax.set_title(f"Linear vs. non-linear models (top {args.n_features} genes)")\n    ax.legend(loc="lower right")\n    fig.tight_layout()\n    fig.savefig(results_dir / "model_comparison.png", dpi=150)\n    plt.close(fig)\n\n\n```\n\n## Step 8: Save all results to disk\nWhy is this step important? In any data science pipeline, generating a plot\nor printing to stdout isn't enough. We serialize the exact configurations, CV\nsplits, and final metrics to a JSON file. This ensures the experiment is fully\nreproducible and can be tracked for future reference.\n\n```python\n    # Compile the final statistics and metadata\n    summary = {\n        "dataset": {\n            "name": "Golub et al. 1999 leukemia gene expression (ALL vs AML)",\n            "n_samples": int(X.shape[0]),\n            "n_genes_total": int(X.shape[1]),\n            "n_genes_selected_per_fold": args.n_features,\n            "class_counts": {"ALL": int(np.sum(y == 0)), "AML": int(np.sum(y == 1))},\n        },\n        "cv": {"n_splits": args.n_splits, "random_state": args.random_state},\n        "majority_class_baseline_accuracy": {\n            "mean": float(dummy_scores.mean()), "std": float(dummy_scores.std())\n        },\n        "model_comparison": cv_results,\n    }\n    # Write summary to json on disk\n    with open(results_dir / "metrics.json", "w") as f:\n        json.dump(summary, f, indent=2)\n\n    print(f"\nSaved plots and metrics.json to: {results_dir.resolve()}")\n\n```\n\n## Step 9: Final Conclusion\n\n```python\n    print("\n================ FINAL CONCLUSION ================")\n    print("This pipeline successfully demonstrated that non-linearity is not a silver bullet.")\n    print("Unlike Project 1, where a linear model failed on 2D synthetic data, here the")\n    print("linear models (Logistic Regression, Linear SVM) outperformed the neural network (MLP).")\n    print("Because we had 3,571 genes but only 72 patients (p >> n), the neural network's")\n    print("extra capacity caused it to overfit. Always match model complexity to your data!")\n    print("==================================================\n")\n\nif __name__ == "__main__":\n    main()\n```\n
